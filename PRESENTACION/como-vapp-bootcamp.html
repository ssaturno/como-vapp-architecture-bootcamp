El problema

¿Qué necesitábamos construir?

Sistema de pedidos de comida con 3 actores independientes que deben comunicarse sin acoplamiento.

🛒
Cliente
Crea pedidos y consulta el estado en tiempo real desde el frontend estático en S3.
👨‍🍳
Cocina / Admin
Gestiona y actualiza el estado de los pedidos. Solo acceso interno — sin exposición pública.
🔔
Notificaciones
Informa al cliente de cada cambio de estado. Consume la cola SQS y envía email vía SES.

Requisitos clave

  • ✓Servicios independientes — fallos aislados, despliegue autónomo sin afectar a los demás
  • ✓Comunicación asíncrona — un servicio caído no bloquea al resto (buffer SQS de 14 días)
  • ✓Trazabilidad completa — cada pedido registrado en DynamoDB con TTL, PITR y Streams habilitados
  • ✓Infraestructura reproducible — terraform destroy + apply = mismo sistema en minutos

Resiliencia

Self-healing: el sistema se repara solo

♻️
Kubernetes reconciliation loop
El control-plane compara continuamente el estado actual vs. el deseado. Con 2 réplicas, si cae 1 pod, K8s crea otro automáticamente. Demo: kubectl delete pod -l app=notifications-service → pods vuelven en ~10 segundos.
📬
SQS como buffer — Zero message loss
Mensajes persisten hasta 14 días. Si notifications-svc cae completamente, los mensajes esperan en la cola y se procesan cuando el servicio vuelve. La DLQ captura los que fallan 3 veces y dispara una CloudWatch Alarm.
🔄
Rolling Update — cero downtime
Estrategia: maxUnavailable=1, maxSurge=1. Siempre al menos 1 pod activo durante el deploy. Rollback instantáneo: kubectl rollout undo deployment/orders-service.
📡
Multi-AZ con topologySpreadConstraints
Las 2 réplicas de cada servicio se distribuyen obligatoriamente entre us-east-1a y us-east-1b. Si una AZ falla, el servicio sigue operativo desde la otra zona.

AWS Academy vs. Producción Real

Decisiones tomadas por limitaciones de LabRole → plan claro de migración a producción

☁️ AWS Academy — implementado hoy 🚀 Producción Real — próximo paso

Credenciales en ConfigMap (base64) AWS Secrets Manager + CSI Driver — secret-provider-class.yaml ya preparado

Compliance continuo

AWS Config — 4 reglas de compliance activas

Monitoreo continuo del estado de seguridad. Cualquier violación queda registrada automáticamente y puede disparar remediación.

🔒
restricted-ssh
Ningún Security Group permite el puerto 22 (SSH) abierto al mundo (0.0.0.0/0 o ::/0). Previene acceso SSH no autorizado desde Internet. Los nodos EKS están en subnets privadas de todas formas.
📡
vpc-flow-logs-enabled
Verifica que VPC Flow Logs estén activos. Permite auditar todo el tráfico de red entrante y saliente para forensics, investigación de incidentes y detección de anomalías.
🔐
encrypted-volumes
Todos los volúmenes EBS de los nodos EKS deben estar cifrados en reposo. Garantiza que los datos almacenados en disco (imágenes de contenedor, logs) nunca estén en texto plano.
🗄️
dynamodb-encryption
La tabla DynamoDB debe tener SSE activado. Garantiza que los datos de pedidos (órdenes, precios, clientes) estén cifrados en reposo y cumplan con regulaciones de protección de datos.

Lecciones aprendidas

5 problemas reales → 5 soluciones reales

⚠ Problema EKS Auto Mode seleccionó instancias ARM — imágenes x86 fallaban al iniciar → ✓ Solución Desactivar Auto Mode → "Configure manually" con tipo de instancia x86_64
⚠ Problema t3.small sin headroom: máx 11 pods, no cabían Prometheus ni Grafana → ✓ Solución Migrar a t3.medium → capacidad para 17 pods por nodo con headroom de observabilidad
⚠ Problema IMDS hop limit=1: pods dentro de K8s no podían obtener credenciales de AWS (LabRole) → ✓ Solución aws ec2 modify-instance-metadata-options --http-put-response-hop-limit 2 en cada nodo
⚠ Problema AWS Load Balancer Controller en CrashLoopBackOff — faltaba región y VPC ID → ✓ Solución Agregar --set region=us-east-1 --set vpcId=vpc-xxx al helm install
⚠ Problema SES bloqueado en AWS Academy — imposible verificar sender para enviar emails reales → ✓ Solución Variable USE_SES=false — notificaciones simuladas en logs. En cuenta real: SES activo con sender verificado.
